# Interview Briefing

## Purpose

Prepare a spokesperson for a media interview with focused messages and likely
questions.

## Default Pairing

- default speaker: James Whitmore
- default media context: BBC talkshow when the request is for a BBC broadcast

## Required Intake

- spokesperson
- outlet or media type
- interview topic
- core messages
- likely critical questions
- no-go areas

## Output Format

```text
INTERVIEW BRIEFING - CONFIDENTIAL
[Spokesperson] | [Outlet] | [Date]
***

1. CONTEXT
[Why the interview happens now]

2. CORE MESSAGES
1. [Message 1]
2. [Message 2]
3. [Message 3]

3. LIKELY QUESTIONS
- [Question 1]
- [Question 2]
- [Question 3]

4. BRIDGING LINES
- [Bridge 1]
- [Bridge 2]

5. DO
- [Practical interview behavior]

6. DO NOT
- [Restricted topic or wording]

7. LOGISTICS
- [time, format, contact]
```

## Rules

- Keep it internal and practical.
- Focus on message discipline, not polished prose.
- Use no more than three core messages.
- Build the core messages from the source material's relevance to BWEC's wind
  energy mandate, not from generic energy commentary.
- If the source material is broad, translate it into implications for wind
  energy, delivery conditions, industrial competitiveness, or energy security.
- Offer bridge lines for difficult questions.
- Do not include unsupported claims or confidential facts.
- Default to James Whitmore unless the user explicitly requests another speaker.
