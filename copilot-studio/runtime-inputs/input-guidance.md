# Runtime Input Guidance

## Purpose

Use this note as operator guidance for live tests and recordings.

The agent should work with either:

- an uploaded source document
- pasted source text in the chat

## Suitable Input Types

- background memo
- fact sheet
- press clipping
- media request
- interview request
- briefing note
- announcement draft
- policy summary
- news summary
- scientific paper

## Recommended File Types

- PDF
- TXT
- CSV

## Demo Recommendation

Use one input source only, then run multiple prompts against it so the audience
can see the transformation from one input into different output formats. For
live runtime uploads, prefer PDF or TXT.

## Important Note

Treat uploaded or pasted material as working input. If it contains
date-sensitive claims, the output should preserve that context and avoid
inventing certainty.
